Git Revert: How to Undo Changes in Git

Git is a powerful and popular version control system that allows you to track changes in your code and collaborate with other developers. However, sometimes you may make a mistake or change your mind about a commit and want to undo it. How can you do that in Git?



One of the most common ways to undo changes in Git is to use the git revert command. This command takes a commit and creates a new commit that applies the inverse of the changes introduced by the original commit. In other words, it reverses the effect of the original commit without deleting it from the history.



Why use git revert?

There are several reasons why you may want to use git revert instead of other undo commands like git reset or git checkout. Here are some of them:



  • Git revert preserves the history. Unlike git reset, which moves the branch pointer to a different commit and erases the subsequent commits, git revert keeps the original commit in the history and adds a new commit on top of it. This is important for the integrity of your project history and for reliable collaboration with other developers.
  • Git revert is safe and easy. Unlike git checkout, which modifies your working directory and index, git revert only changes the files that were affected by the original commit. This means that you don't have to worry about losing any uncommitted changes or staging them before reverting. Git revert also handles conflicts automatically by creating a merge commit if necessary.
  • Git revert is flexible and powerful. You can use git revert to undo any commit, not just the most recent one. You can also revert multiple commits at once, or revert a merge commit by specifying which parent to use as the mainline. You can also edit the commit message or sign the revert commit with your GPG key.


How to use git revert?

The basic syntax of the git revert command is:



git revert commit-hash

where commit-hash is the reference of the commit that you want to revert. You can use any valid way to specify a commit, such as a hash, a tag, a branch name, or a relative reference. For example, to revert the last commit, you can use:



git revert HEAD

To revert the second-to-last commit, you can use:



git revert HEAD~1

To revert a specific commit by its hash, you can use:



git revert 86bb32e

By default, git revert will open your editor and let you edit the commit message before creating the revert commit. The commit message will start with "Revert" followed by the original commit message. You can modify the message as you like, or keep it as it is. To skip the editor and use the default message, you can use the --no-edit option:



git revert --no-edit

To revert multiple commits at once, you can list them in the command separated by spaces. For example, to revert the last three commits, you can use:



git revert HEAD HEAD~1 HEAD~2

Note that the order of the commits matters. Git will revert them in the reverse order that you specify them. In the example above, git will first revert HEAD~2, then HEAD~1, then HEAD. This is to ensure that the revert commits are applied in the same order as the original commits.



To revert a merge commit, you need to use the -m or --mainline option to tell git which parent of the merge to use as the mainline. For example, to revert a merge commit with two parents, you can use:



git revert -m 1

This will revert the changes that were introduced by merging the second parent into the first parent. To revert the changes that were introduced by merging the first parent into the second parent, you can use:



git revert -m 2

Reverting a merge commit is a serious operation, as it declares that you will never want the changes from the other branch. This may cause conflicts or unexpected results in future merges. Therefore, you should only revert a merge commit if you are sure that you don't need the changes from the other branch.



Conclusion

Git revert is a useful command that allows you to undo changes in your project history without losing or modifying the original commits. It creates a new commit that applies the inverse of the changes from the original commit, preserving the history and making it easy to collaborate with other developers. You can use git revert to undo any commit, multiple commits, or merge commits, with various options and flags to customize the behavior.
